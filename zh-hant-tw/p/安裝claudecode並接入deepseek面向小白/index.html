概述

這份教學是給誰看的？

這份教學面向完全沒有電腦基礎的使用者。你不需要知道什麼是「命令列」、什麼是「環境變數」、什麼是「版本管理」——我會把每一步都拆解到最細，告訴你在哪裡點、點什麼、輸入什麼、看到什麼說明成功了。

你要裝什麼？為什麼要裝這些？

在開始之前，先瞭解我們一共要安裝 3 個東西，它們各自的作用：

序號 軟體名稱 一句話解釋 為什麼必須裝
1 Node.js 讓你的電腦能執行 JavaScript 程式的「引擎」 Claude Code 是用 JavaScript 寫的，沒有它就跑不起來
2 Claude Code 一個住在你電腦終端機裡的 AI 程式設計助手 這就是我們最終要用的東西
3 CC-Switch 一個給 Claude Code「換大腦」的小工具 讓 Claude Code 從昂貴的官方模型換成便宜的 DeepSeek

整個過程大約需要 10-15 分鐘。請按順序一步步來，不要跳步。

📌 前提條件

你需要一台 Windows 電腦（Windows 10/11），能正常上網。

安裝 Node.js

什麼是 Node.js？

Node.js = 讓電腦能執行 Claude Code 的執行環境。它是完全免費的，就像你需要先裝好 Windows 系統才能執行軟體一樣，你需要先裝好 Node.js 才能執行 Claude Code。

下載 Node.js

第 1 步：開啟瀏覽器

開啟你電腦上任意一個瀏覽器（Chrome、Edge、Firefox 都可以）。

第 2 步：前往 Node.js 官網

在瀏覽器網址列（頂部輸入網址的長條框）中輸入以下網址，然後按 Enter：

1

https://nodejs.org/

第 3 步：找到下載按鈕

開啟網頁後，你會看到頁面中央有兩個大按鈕：

  • 左邊的按鈕寫著 LTS（推薦的版本號，比如 22.16.0 LTS）
  • 右邊的按鈕寫著 Current

點選左邊的 LTS 按鈕，瀏覽器會自動開始下載一個 .msi 檔案。

第 4 步：找到下載好的檔案

下載完成後（通常只需幾秒鐘），有兩種方式找到這個檔案：

  • 方式一：點選瀏覽器右上角的下載列表（通常是一個向下的箭頭 ↓ 圖示），在列表中找到剛才下載的檔案，檔名類似 node-v22.16.0-x64.msi
  • 方式二：開啟檔案總管（就是你看資料夾的那個視窗），在左側找到下載資料夾，裡面就有這個檔案

安裝 Node.js

第 1 步：按兩下安裝檔案

在下載資料夾中，找到 node-v22.16.0-x64.msi（版本號可能略有不同），按兩下它。

第 2 步：歡迎介面

彈出一個標題為 Node.js Setup（Node.js 安裝精靈）的視窗。 點選底部的 Next（下一步）按鈕。

第 3 步：授權合約

視窗中顯示一大段英文合約文字。

  • 勾選左下角的核取方塊：I accept the terms in the License Agreement（我接受授權合約中的條款）
  • 點選 Next

第 4 步：安裝位置（⚠️ 重要！需要修改路徑）

這裡顯示的是 Node.js 將要安裝到你電腦的哪個位置。

預設路徑是：C:\Program Files\nodejs\

📌 我們要把它改到 D 磁碟！

建議安裝到 D 磁碟，原因：

  • C 磁碟是系統磁碟，放太多東西會影響電腦速度
  • D 磁碟專門用來裝軟體，萬一系統重灌了，D 磁碟的東西還在
  • 方便管理和查詢

修改步驟：

  1. 點選路徑輸入框，把裡面的內容全部選中（按 Ctrl + A），然後刪除
  2. 輸入新的路徑： 
    1

    D:\nodejs\
  3. 點選 Next
💡 注意

路徑中不要有中文和空格，否則可能導致安裝出錯。

第 5 步：自訂安裝

顯示一個帶有樹狀核取方塊的頁面（Custom Setup）。

什麼都不要改，直接點選 Next。

第 6 步：確認安裝

點選 Install（安裝）按鈕。如果彈出安全提示，點選 是。

第 7 步：等待安裝

進度條會走完，通常只需 10-30 秒。

第 8 步：完成

點選 Finish（完成）。

Node.js 安裝完成！

此時 Node.js 和 npm（Node.js 的套件管理器，可以理解為「軟體商店」）都已經裝好了。

驗證 Node.js 是否安裝成功

安裝完成後，我們需要驗證一下——就像買完菜要檢查一下有沒有漏掉的東西。

第 1 步：開啟命令提示字元（CMD）

什麼是「命令提示字元」？

命令提示字元（也叫 CMD）是一個黑色背景、白色文字的視窗，你可以在這個視窗裡輸入文字命令來讓電腦執行操作。以後會經常用到它。

開啟方式（任選一種）：

  • 方式一（推薦）：

    1. 按住鍵盤上的 Win 鍵（鍵盤左下角，畫著 Windows 圖示的那個鍵）
    2. 同時按下 R 鍵
    3. 彈出一個小視窗（標題是「執行」），在裡面輸入 cmd
    4. 點選確定，或直接按Enter 鍵

  • 方式二：

    1. 點選桌面左下角的開始功能表（Windows 圖示）
    2. 在搜尋框中輸入 cmd
    3. 點選搜尋結果中的 命令提示字元

第 2 步：輸入驗證命令

在開啟的黑色視窗中，輸入以下內容（輸入時螢幕上會顯示你打的字），然後按Enter 鍵：

1

node -v

如果視窗中顯示類似 v22.16.0 的內容，說明 Node.js 安裝成功了 ✅

繼續輸入以下命令，按 Enter：

1

npm -v

如果顯示類似 10.9.2 的版本號，說明 npm 也正常 ✅

⚠️ 提示「不是內部或外部命令」？

說明環境變數沒有配置好，請看下面的 1.4 如何配置環境變數。

如何配置環境變數

什麼是環境變數（PATH）？

PATH 可以理解為電腦的「通訊錄」。當你在命令提示字元裡輸入 node 時，電腦不知道 node 裝在哪裡，它會去 PATH 這個「通訊錄」裡找。如果 PATH 裡沒有 Node.js 的安裝目錄，電腦就找不到它，就會報「不是內部或外部命令」。

正常情況下，Node.js 安裝時會自動把自己加到 PATH 裡。但如果出了問題，你需要手動新增。

檢視 PATH 裡有沒有 Node.js

  1. 按 Win + R，輸入 sysdm.cpl，Enter
  2. 點選頂部的 進階 索引標籤
  3. 點選底部的 環境變數 按鈕
  4. 在下方 系統變數 列表中，找到 Path 這一行，選中它，點選 編輯

在彈出的列表中，檢查有沒有類似這樣的條目：

1

D:\nodejs\
💡 路徑要和你安裝時設定的一致

如果你當時裝在了 D:\Software\nodejs\，那這裡也應該顯示 D:\Software\nodejs\。

如果有這個條目：說明 PATH 沒問題，但仍然報錯的話，關掉當前命令提示字元視窗，重新開啟一個新的再試。

如果沒有這個條目：按下面步驟手動新增。

手動新增 Node.js 到 PATH

  1. 在剛才開啟的 編輯環境變數 視窗中，點選 新增
  2. 輸入 Node.js 的安裝路徑： 
    1

    D:\nodejs\
  3. 點選 確定
  4. 點選 確定 關閉環境變數視窗
  5. 點選 確定 關閉系統內容視窗
  6. 關掉所有已開啟的命令提示字元視窗，重新開啟一個新的（舊視窗不會識別新配置）
  7. 再次輸入 node -v 驗證

配置 npm 國內鏡像（強烈推薦）

💡 為什麼要配置？

npm 預設從國外下載軟體，速度很慢。配置國內鏡像後下載速度提升 10 倍以上。

在命令提示字元中輸入以下命令，按 Enter：

1

npm config set registry https://registry.npmmirror.com/

輸入以下命令驗證，如果顯示 https://registry.npmmirror.com/ 則成功 ✅：

1

npm config get registry
第一步完成！

你已經在電腦上安裝好了 Node.js，並且配置了國內鏡像。接下來安裝 Claude Code。

安裝 Claude Code

透過 npm 安裝 Claude Code

開啟命令提示字元（如果還沒開啟的話），輸入以下命令，按 Enter：

1

npm install -g @anthropic-ai/claude-code

等待安裝完成（一般 1-3 分鐘）。螢幕上會捲動一些文字，只要最後沒有報錯就行。

驗證安裝

輸入：

1

claude --version

按 Enter。如果顯示版本號（例如 1.0.x），說明安裝成功 ✅

首次啟動（跳過登入）

為什麼要跳過登入？

Claude Code 官方要求你用 Anthropic 帳號登入才能使用。但 Anthropic 的服務需要海外信用卡支付，對國內使用者很不方便。

我們的方案是：先跳過登入，然後透過 CC-Switch 工具接入國產的 DeepSeek 模型，這樣就不需要 Anthropic 帳號了，費用也低得多。

第 1 步：先啟動一次 Claude Code

在命令提示字元中輸入：

1

claude

按 Enter。Claude Code 會啟動，可能會顯示一些介紹資訊或要求你登入。

不用管它，直接關閉這個命令提示字元視窗（點右上角的 ×）。

第 2 步：找到配置檔案

  1. 按 Win + R（Win 鍵 + R 鍵同時按）
  2. 在彈出的「執行」視窗中，輸入 %USERPROFILE%
  3. 點選確定或按 Enter
  4. 開啟後，你會看到一個資料夾，裡面有各種檔案和子資料夾

第 3 步：找到 .claude.json 檔案

在這個資料夾中，找到一個名為 .claude.json 的檔案。

⚠️ 找不到這個檔案？

如果找不到 .claude.json，可能是以下原因：

  • 副檔名被隱藏了——在資料夾視窗頂部，點選 檢視 → 勾選 副檔名 和 隱藏的專案
  • Claude Code 還沒有建立這個檔案——請回到第 1 步，確保你成功啟動過一次 Claude Code
  • 在某些情況下，這個檔案可能在 .claude 資料夾裡面，檔名為 settings.json

第 4 步：用記事本開啟配置檔案

  1. 右鍵點選 .claude.json 檔案
  2. 在彈出的選單中，選擇 開啟方式
  3. 選擇 記事本（Notepad）
    • 如果列表中沒有記事本，選擇 選擇其他應用 → 在列表中找到 記事本 → 點選 確定

第 5 步：編輯配置檔案

開啟後，你會看到類似這樣的內容（具體內容可能不同）：

1
2
3
4

{
  "someField": "someValue",
  "anotherField": "anotherValue"
}

我們需要在最後的 } 前面，新增一行 "hasCompletedOnboarding": true。

修改後的樣子應該是這樣的：

1
2
3
4
5

{
  "someField": "someValue",
  "anotherField": "anotherValue",
  "hasCompletedOnboarding": true
}
📌 ⚠️ 注意三個細節，否則會報錯！
  • 加逗號：hasCompletedOnboarding 前面那行的末尾要加一個英文逗號 ,
  • 用英文引號：引號必須是英文的 "，不能是中文的 "
  • true 是小寫：必須寫 true，不能寫 True

第 6 步：儲存檔案

  1. 按 Ctrl + S 儲存（或者點選左上角 檔案 → 儲存）
  2. 關閉記事本視窗

第 7 步：驗證跳過登入是否成功

  1. 重新開啟命令提示字元（Win + R → cmd → Enter）
  2. 輸入 claude，按 Enter
  3. 如果直接進入了 Claude Code 的主介面（沒有要求你登入），說明跳過登入成功 ✅
第二步完成！

Claude Code 安裝完畢，並且跳過了登入步驟。接下來安裝 CC-Switch 來接入 DeepSeek。

安裝 CC-Switch（模型切換工具）

什麼是 CC-Switch？

CC-Switch 是一個視覺化工具，讓你不用手動改配置檔案，點點滑鼠就能把 Claude Code 的「大腦」從官方模型換成 DeepSeek。

下載 CC-Switch

第 1 步：開啟下載頁面

在瀏覽器網址列中輸入以下網址，按 Enter：

1

https://github.com/cc-switch/cc-switch/releases

第 2 步：找到最新版本

開啟頁面後，找到帶有綠色 Latest 標籤的版本，點選展開。

第 3 步：找到下載連結

展開後往下捲動，找到 Assets 這個詞（旁邊可能有個小箭頭 ▾），點選展開它。

找到以 .msi 結尾的檔案（Windows 安裝包），點選檔名開始下載。

💡 選哪個檔案？
  • Windows 電腦：選 x64.msi 結尾的檔案
  • Mac 電腦：選 .dmg 結尾的檔案

安裝 CC-Switch

第 1 步：按兩下安裝檔案

在下載資料夾中，按兩下下載好的 .msi 檔案。如果彈出安全提示，點選 仍要執行 或 更多資訊 → 仍要執行。

第 2 步：按提示安裝

安裝精靈和 Node.js 類似：

  1. 點選 Next
  2. 接受合約 → Next
  3. 保持預設安裝路徑 → Next
  4. 點選 Install
  5. 等待安裝完成 → Finish
第三步完成！

CC-Switch 安裝好了。接下來獲取 DeepSeek 的 API Key。

獲取 DeepSeek API Key

什麼是 DeepSeek？

國產大語言模型，效能強大且價格低廉（日常使用每月約 10-30 元），接入後可以大幅降低使用成本。

什麼是 API Key？

API Key 是一把鑰匙，用來呼叫 DeepSeek 的 AI 服務，每次使用會從帳戶餘額中扣費。你需要：① 註冊帳號 ② 充值 ③ 建立 Key

註冊 DeepSeek 帳號

第 1 步：開啟 DeepSeek 開放平臺

在瀏覽器網址列中輸入：

1

https://platform.deepseek.com/

按 Enter。

💡 注意

這是 DeepSeek 的開發者平臺（platform.deepseek.com），不是聊天網站（chat.deepseek.com），別搞混了。

第 2 步：註冊/登入

  1. 頁面右上角有 登入 / 註冊 按鈕
  2. 點選 註冊（如果沒有帳號）或 登入（已有帳號）
  3. 按頁面提示填寫資訊完成註冊

充值餘額

💡 費用參考

日常使用一個月大約只需 10-30 元，建議首次充值 20-50 元。

  1. 登入後，在頁面左側選單中找到 充值（或 錢包 / Billing），點選它
  2. 選擇充值金額，選擇支付方式，完成支付
  3. 確保帳戶餘額大於 0（沒有餘額無法使用 API）

建立 API Key

第 1 步：進入 API Keys 頁面

  1. 登入後，在左側選單中找到 API Keys，點選它
  2. 你會看到一個 API Keys 管理頁面

第 2 步：建立新的 API Key

  1. 點選 建立 API Key 按鈕
  2. 彈出一個對話方塊，要求你輸入 Key 的名稱
    • 可以隨意填寫，比如 claude-code、我的key、test 都行
    • 這個名稱只是給你自己看的，用來區分不同的 Key
  3. 點選 確認 或 建立

第 3 步：複製 API Key（⚠️ 最關鍵的一步！）

建立成功後，頁面會顯示你的 API Key，格式類似：

1

sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
⚠️ 這個 Key 只會顯示一次！

關閉對話方塊後就再也看不到了。請立即複製儲存到記事本或微信檔案傳輸助手。

🚨 安全提醒

API Key 等同於你的帳號密碼，不要分享給任何人，不要發到公開場合。如果懷疑洩露，立即刪除並重新建立。

第四步完成！

你已經有了 DeepSeek 的 API Key（鑰匙）。最後一步，把這把鑰匙交給 CC-Switch，完成接入。

配置 CC-Switch 接入 DeepSeek

開啟 CC-Switch

在桌面上或開始選單中找到 CC-Switch 的圖示，按兩下開啟。

新增 DeepSeek 配置

第 1 步：點選新增按鈕

CC-Switch 開啟後，找到右上角的 + 號（加號按鈕），點選它。

第 2 步：填寫配置資訊

點選 + 號後，會彈出一個配置面板，填寫以下資訊：

① 模型預設（Model Preset）：選擇 DeepSeek

② API Key：貼上你之前複製的 DeepSeek API Key（注意首尾不要有空格）

③ 模型名稱（Model Name）：把輸入框內容刪掉，輸入：

1

deepseek-v4-pro[1m]

④ Base URL / API 位址（如果有的話）：

1

https://api.deepseek.com

第 3 步：儲存配置

檢查所有資訊都填好後，點選 新增 / 儲存 / 確認 按鈕。

切換到 DeepSeek

  1. 回到 CC-Switch 的主介面
  2. 你會看到剛才新增的 DeepSeek 配置已經出現在列表中
  3. 點選該配置旁邊的 切換 / 啟用 按鈕
  4. 介面上應該會顯示「當前模型：DeepSeek」或類似的資訊

驗證接入是否成功

開啟命令提示字元，輸入 claude 啟動 Claude Code。

在 Claude Code 中輸入：

1

/model

按 Enter。如果顯示的模型資訊中包含 DeepSeek，說明接入成功 ✅

全部配置完成！

現在可以開始使用 AI 程式設計助手了！

開始使用（實戰範例）

配置完成後，做一個簡單的練習確保一切正常。

建立專案資料夾並開啟命令提示字元

  1. 在桌面右鍵 → 新增 → 資料夾，命名為 my-first-ai-project
  2. 按兩下開啟這個資料夾
  3. 點選視窗頂部的網址列，輸入 cmd，按 Enter

在彈出的命令提示字元視窗中，輸入：

常見問題 FAQ

輸入 node -v 提示「不是內部或外部命令」？

Node.js 的環境變數（PATH）沒有配置好。開啟檔案總管，進入 D:\nodejs\ 看看裡面有沒有 node.exe：

  • 有這個檔案：說明安裝了但 PATH 沒配好，按照上方 1.4 如何配置環境變數 的步驟手動新增
  • 沒有這個檔案：說明安裝失敗了，重新下載安裝
輸入 claude 提示「不是內部或外部命令」？

Claude Code 安裝沒成功。先確認 node -v 和 npm -v 正常，然後重新執行 npm install -g @anthropic-ai/claude-code。

啟動 Claude Code 後要求登入 Anthropic 帳號？

.claude.json 沒改對。按 Win + R，輸入 %USERPROFILE% Enter，找到 .claude.json 用記事本開啟，確認裡面有 "hasCompletedOnboarding": true，且前面那行末尾有英文逗號、true 是小寫、引號是英文引號。

Claude Code 對話時報錯 / 無回應？

常見原因：① DeepSeek 餘額不足 ② API Key 填寫錯誤 ③ 模型名稱不是 deepseek-v4-pro[1m] ④ 網路問題。逐項檢查即可。

npm 安裝 Claude Code 非常慢（超過 5 分鐘）？

檢查是否配置了國內鏡像：npm config get registry（應顯示 registry.npmmirror.com）。如果沒有，執行 npm config set registry https://registry.npmmirror.com/ 後重試。

CC-Switch 打不開或閃退？

到 GitHub Release 重新下載最新版本，或嘗試右鍵以系統管理員身份執行。

GitHub 頁面打不開？

原因：GitHub 在國內可能訪問不穩定。 解決：

  • 多重新整理幾次頁面
  • 嘗試使用不同的瀏覽器
  • 如果實在打不開，可以在網上搜尋「CC-Switch 下載」，有些國內鏡像站也提供了下載
