概述

这份教程是给谁看的？

这份教程面向完全没有计算机基础的用户。你不需要知道什么是"命令行"、什么是"环境变量"、什么是"版本管理"——我会把每一步都拆解到最细，告诉你在哪里点、点什么、输入什么、看到什么说明成功了。

你要装什么？为什么要装这些？

在开始之前，先了解我们一共要安装 3 个东西，它们各自的作用：

序号 软件名称 一句话解释 为什么必须装
1 Node.js 让你的电脑能运行 JavaScript 程序的"引擎" Claude Code 是用 JavaScript 写的，没有它就跑不起来
2 Claude Code 一个住在你电脑终端里的 AI 编程助手 这就是我们最终要用的东西
3 CC-Switch 一个给 Claude Code “换大脑"的小工具 让 Claude Code 从昂贵的官方模型换成便宜的 DeepSeek

整个过程大约需要 10-15 分钟。请按顺序一步步来，不要跳步。

📌 前提条件

你需要一台 Windows 电脑（Windows 10/11），能正常上网。

安装 Node.js

什么是 Node.js？

Node.js = 让电脑能运行 Claude Code 的运行环境。它是完全免费的，就像你需要先装好 Windows 系统才能运行软件一样，你需要先装好 Node.js 才能运行 Claude Code。

下载 Node.js

第 1 步：打开浏览器

打开你电脑上任意一个浏览器（Chrome、Edge、Firefox 都可以）。

第 2 步：访问 Node.js 官网

在浏览器地址栏（顶部输入网址的长条框）中输入以下网址，然后按回车：

1

https://nodejs.org/

第 3 步：找到下载按钮

打开网页后，你会看到页面中央有两个大按钮：

  • 左边的按钮写着 LTS（推荐的版本号，比如 22.16.0 LTS）
  • 右边的按钮写着 Current

点击左边的 LTS 按钮，浏览器会自动开始下载一个 .msi 文件。

第 4 步：找到下载好的文件

下载完成后（通常只需几秒钟），有两种方式找到这个文件：

  • 方式一：点击浏览器右上角的下载列表（通常是一个向下的箭头 ↓ 图标），在列表中找到刚才下载的文件，文件名类似 node-v22.16.0-x64.msi
  • 方式二：打开文件资源管理器（就是你看文件夹的那个窗口），在左侧找到下载文件夹，里面就有这个文件

安装 Node.js

第 1 步：双击安装文件

在下载文件夹中，找到 node-v22.16.0-x64.msi（版本号可能略有不同），双击它。

第 2 步：欢迎界面

弹出一个标题为 Node.js Setup（Node.js 安装向导）的窗口。 点击底部的 Next（下一步）按钮。

第 3 步：许可协议

窗口中显示一大段英文协议文字。

  • 勾选左下角的复选框：I accept the terms in the License Agreement（我接受许可协议中的条款）
  • 点击 Next

第 4 步：安装位置（⚠️ 重要！需要修改路径）

这里显示的是 Node.js 将要安装到你电脑的哪个位置。

默认路径是：C:\Program Files\nodejs\

📌 我们要把它改到 D 盘！

建议安装到 D 盘，原因：

  • C 盘是系统盘，放太多东西会影响电脑速度
  • D 盘专门用来装软件，万一系统重装了，D 盘的东西还在
  • 方便管理和查找

修改步骤：

  1. 点击路径输入框，把里面的内容全部选中（按 Ctrl + A），然后删除
  2. 输入新的路径： 
    1

    D:\nodejs\
  3. 点击 Next
💡 注意

路径中不要有中文和空格，否则可能导致安装出错。

第 5 步：自定义安装

显示一个带有树状复选框的页面（Custom Setup）。

什么都不要改，直接点击 Next。

第 6 步：确认安装

点击 Install（安装）按钮。如果弹出安全提示，点击 是。

第 7 步：等待安装

进度条会走完，通常只需 10-30 秒。

第 8 步：完成

点击 Finish（完成）。

Node.js 安装完成！

此时 Node.js 和 npm（Node.js 的包管理器，可以理解为"软件商店”）都已经装好了。

验证 Node.js 是否安装成功

安装完成后，我们需要验证一下——就像买完菜要检查一下有没有漏掉的东西。

第 1 步：打开命令提示符（CMD）

什么是"命令提示符"？

命令提示符（也叫 CMD）是一个黑色背景、白色文字的窗口，你可以在这个窗口里输入文字命令来让电脑执行操作。以后会经常用到它。

打开方式（任选一种）：

  • 方式一（推荐）：

    1. 按住键盘上的 Win 键（键盘左下角，画着 Windows 图标的那个键）
    2. 同时按下 R 键
    3. 弹出一个小窗口（标题是"运行"），在里面输入 cmd
    4. 点击确定，或直接按回车键（Enter）

  • 方式二：

    1. 点击桌面左下角的开始菜单（Windows 图标）
    2. 在搜索框中输入 cmd
    3. 点击搜索结果中的 命令提示符

第 2 步：输入验证命令

在打开的黑色窗口中，输入以下内容（输入时屏幕上会显示你打的字），然后按回车键：

1

node -v

如果窗口中显示类似 v22.16.0 的内容，说明 Node.js 安装成功了 ✅

继续输入以下命令，按回车：

1

npm -v

如果显示类似 10.9.2 的版本号，说明 npm 也正常 ✅

⚠️ 提示"不是内部或外部命令"？

说明环境变量没有配置好，请看下面的 1.4 如何配置环境变量。

如何配置环境变量

什么是环境变量（PATH）？

PATH 可以理解为电脑的"通讯录"。当你在命令提示符里输入 node 时，电脑不知道 node 装在哪里，它会去 PATH 这个"通讯录"里找。如果 PATH 里没有 Node.js 的安装目录，电脑就找不到它，就会报"不是内部或外部命令"。

正常情况下，Node.js 安装时会自动把自己加到 PATH 里。但如果出了问题，你需要手动添加。

查看 PATH 里有没有 Node.js

  1. 按 Win + R，输入 sysdm.cpl，回车
  2. 点击顶部的 高级 选项卡
  3. 点击底部的 环境变量 按钮
  4. 在下方 系统变量 列表中，找到 Path 这一行，选中它，点击 编辑

在弹出的列表中，检查有没有类似这样的条目：

1

D:\nodejs\
💡 路径要和你安装时设置的一致

如果你当时装在了 D:\Software\nodejs\，那这里也应该显示 D:\Software\nodejs\。

如果有这个条目：说明 PATH 没问题，但仍然报错的话，关掉当前命令提示符窗口，重新打开一个新的再试。

如果没有这个条目：按下面步骤手动添加。

手动添加 Node.js 到 PATH

  1. 在刚才打开的 编辑环境变量 窗口中，点击 新建
  2. 输入 Node.js 的安装路径： 
    1

    D:\nodejs\
  3. 点击 确定
  4. 点击 确定 关闭环境变量窗口
  5. 点击 确定 关闭系统属性窗口
  6. 关掉所有已打开的命令提示符窗口，重新打开一个新的（旧窗口不会识别新配置）
  7. 再次输入 node -v 验证

配置 npm 国内镜像（强烈推荐）

💡 为什么要配置？

npm 默认从国外下载软件，速度很慢。配置国内镜像后下载速度提升 10 倍以上。

在命令提示符中输入以下命令，按回车：

1

npm config set registry https://registry.npmmirror.com/

输入以下命令验证，如果显示 https://registry.npmmirror.com/ 则成功 ✅：

1

npm config get registry
第一步完成！

你已经在电脑上安装好了 Node.js，并且配置了国内镜像。接下来安装 Claude Code。

安装 Claude Code

通过 npm 安装 Claude Code

打开命令提示符（如果还没打开的话），输入以下命令，按回车：

1

npm install -g @anthropic-ai/claude-code

等待安装完成（一般 1-3 分钟）。屏幕上会滚动一些文字，只要最后没有报错就行。

验证安装

输入：

1

claude --version

按回车。如果显示版本号（例如 1.0.x），说明安装成功 ✅

首次启动（跳过登录）

为什么要跳过登录？

Claude Code 官方要求你用 Anthropic 账号登录才能使用。但 Anthropic 的服务需要海外信用卡支付，对国内用户很不方便。

我们的方案是：先跳过登录，然后通过 CC-Switch 工具接入国产的 DeepSeek 模型，这样就不需要 Anthropic 账号了，费用也低得多。

第 1 步：先启动一次 Claude Code

在命令提示符中输入：

1

claude

按回车。Claude Code 会启动，可能会显示一些介绍信息或要求你登录。

不用管它，直接关闭这个命令提示符窗口（点右上角的 ×）。

第 2 步：找到配置文件

  1. 按 Win + R（Win 键 + R 键同时按）
  2. 在弹出的"运行"窗口中，输入 %USERPROFILE%
  3. 点击确定或按回车
  4. 打开后，你会看到一个文件夹，里面有各种文件和子文件夹

第 3 步：找到 .claude.json 文件

在这个文件夹中，找到一个名为 .claude.json 的文件。

⚠️ 找不到这个文件？

如果找不到 .claude.json，可能是以下原因：

  • 文件扩展名被隐藏了——在文件夹窗口顶部，点击 查看 → 勾选 文件扩展名 和 隐藏的项目
  • Claude Code 还没有创建这个文件——请回到第 1 步，确保你成功启动过一次 Claude Code
  • 在某些情况下，这个文件可能在 .claude 文件夹里面，文件名为 settings.json

第 4 步：用记事本打开配置文件

  1. 右键点击 .claude.json 文件
  2. 在弹出的菜单中，选择 打开方式
  3. 选择 记事本（Notepad）
    • 如果列表中没有记事本，选择 选择其他应用 → 在列表中找到 记事本 → 点击 确定

第 5 步：编辑配置文件

打开后，你会看到类似这样的内容（具体内容可能不同）：

1
2
3
4

{
  "someField": "someValue",
  "anotherField": "anotherValue"
}

我们需要在最后的 } 前面，添加一行 "hasCompletedOnboarding": true。

修改后的样子应该是这样的：

1
2
3
4
5

{
  "someField": "someValue",
  "anotherField": "anotherValue",
  "hasCompletedOnboarding": true
}
📌 ⚠️ 注意三个细节，否则会报错！
  • 加逗号：hasCompletedOnboarding 前面那行的末尾要加一个英文逗号 ,
  • 用英文引号：引号必须是英文的 "，不能是中文的 "
  • true 是小写：必须写 true，不能写 True

第 6 步：保存文件

  1. 按 Ctrl + S 保存（或者点击左上角 文件 → 保存）
  2. 关闭记事本窗口

第 7 步：验证跳过登录是否成功

  1. 重新打开命令提示符（Win + R → cmd → 回车）
  2. 输入 claude，按回车
  3. 如果直接进入了 Claude Code 的主界面（没有要求你登录），说明跳过登录成功 ✅
第二步完成！

Claude Code 安装完毕，并且跳过了登录步骤。接下来安装 CC-Switch 来接入 DeepSeek。

安装 CC-Switch（模型切换工具）

什么是 CC-Switch？

CC-Switch 是一个可视化工具，让你不用手动改配置文件，点点鼠标就能把 Claude Code 的"大脑"从官方模型换成 DeepSeek。

下载 CC-Switch

第 1 步：打开下载页面

在浏览器地址栏中输入以下网址，按回车：

1

https://github.com/cc-switch/cc-switch/releases

第 2 步：找到最新版本

打开页面后，找到带有绿色 Latest 标签的版本，点击展开。

第 3 步：找到下载链接

展开后往下滚动，找到 Assets 这个词（旁边可能有个小箭头 ▾），点击展开它。

找到以 .msi 结尾的文件（Windows 安装包），点击文件名开始下载。

💡 选哪个文件？
  • Windows 电脑：选 x64.msi 结尾的文件
  • Mac 电脑：选 .dmg 结尾的文件

安装 CC-Switch

第 1 步：双击安装文件

在下载文件夹中，双击下载好的 .msi 文件。如果弹出安全提示，点击 仍要运行 或 更多信息 → 仍要运行。

第 2 步：按提示安装

安装向导和 Node.js 类似：

  1. 点击 Next
  2. 接受协议 → Next
  3. 保持默认安装路径 → Next
  4. 点击 Install
  5. 等待安装完成 → Finish
第三步完成！

CC-Switch 安装好了。接下来获取 DeepSeek 的 API Key。

获取 DeepSeek API Key

什么是 DeepSeek？

国产大语言模型，性能强大且价格低廉（日常使用每月约 10-30 元），接入后可以大幅降低使用成本。

什么是 API Key？

API Key 是一把钥匙，用来调用 DeepSeek 的 AI 服务，每次使用会从账户余额中扣费。你需要：① 注册账号 ② 充值 ③ 创建 Key

注册 DeepSeek 账号

第 1 步：打开 DeepSeek 开放平台

在浏览器地址栏中输入：

1

https://platform.deepseek.com/

按回车。

💡 注意

这是 DeepSeek 的开发者平台（platform.deepseek.com），不是聊天网站（chat.deepseek.com），别搞混了。

第 2 步：注册/登录

  1. 页面右上角有 登录 / 注册 按钮
  2. 点击 注册（如果没有账号）或 登录（已有账号）
  3. 按页面提示填写信息完成注册

充值余额

💡 费用参考

日常使用一个月大约只需 10-30 元，建议首次充值 20-50 元。

  1. 登录后，在页面左侧菜单中找到 充值（或 钱包 / Billing），点击它
  2. 选择充值金额，选择支付方式，完成支付
  3. 确保账户余额大于 0（没有余额无法使用 API）

创建 API Key

第 1 步：进入 API Keys 页面

  1. 登录后，在左侧菜单中找到 API Keys，点击它
  2. 你会看到一个 API Keys 管理页面

第 2 步：创建新的 API Key

  1. 点击 创建 API Key 按钮
  2. 弹出一个对话框，要求你输入 Key 的名称
    • 可以随意填写，比如 claude-code、我的key、test 都行
    • 这个名称只是给你自己看的，用来区分不同的 Key
  3. 点击 确认 或 创建

第 3 步：复制 API Key（⚠️ 最关键的一步！）

创建成功后，页面会显示你的 API Key，格式类似：

1

sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
⚠️ 这个 Key 只会显示一次！

关闭对话框后就再也看不到了。请立即复制保存到记事本或微信文件传输助手。

🚨 安全提醒

API Key 等同于你的账号密码，不要分享给任何人，不要发到公开场合。如果怀疑泄露，立即删除并重新创建。

第四步完成！

你已经有了 DeepSeek 的 API Key（钥匙）。最后一步，把这把钥匙交给 CC-Switch，完成接入。

配置 CC-Switch 接入 DeepSeek

打开 CC-Switch

在桌面上或开始菜单中找到 CC-Switch 的图标，双击打开。

添加 DeepSeek 配置

第 1 步：点击添加按钮

CC-Switch 打开后，找到右上角的 + 号（加号按钮），点击它。

第 2 步：填写配置信息

点击 + 号后，会弹出一个配置面板，填写以下信息：

① 模型预设（Model Preset）：选择 DeepSeek

② API Key：粘贴你之前复制的 DeepSeek API Key（注意首尾不要有空格）

③ 模型名称（Model Name）：把输入框内容删掉，输入：

1

deepseek-v4-pro[1m]

④ Base URL / API 地址（如果有的话）：

1

https://api.deepseek.com

第 3 步：保存配置

检查所有信息都填好后，点击 添加 / 保存 / 确认 按钮。

切换到 DeepSeek

  1. 回到 CC-Switch 的主界面

常见问题 FAQ

输入 node -v 提示"不是内部或外部命令"？

Node.js 的环境变量（PATH）没有配置好。打开文件资源管理器，进入 D:\nodejs\ 看看里面有没有 node.exe：

  • 有这个文件：说明安装了但 PATH 没配好，按照上方 1.4 如何配置环境变量 的步骤手动添加
  • 没有这个文件：说明安装失败了，重新下载安装
输入 claude 提示"不是内部或外部命令"？

Claude Code 安装没成功。先确认 node -v 和 npm -v 正常，然后重新执行 npm install -g @anthropic-ai/claude-code。

启动 Claude Code 后要求登录 Anthropic 账号？

.claude.json 没改对。按 Win + R，输入 %USERPROFILE% 回车，找到 .claude.json 用记事本打开，确认里面有 "hasCompletedOnboarding": true，且前面那行末尾有英文逗号、true 是小写、引号是英文引号。

Claude Code 对话时报错 / 无响应？

常见原因：① DeepSeek 余额不足 ② API Key 填写错误 ③ 模型名称不是 deepseek-v4-pro[1m] ④ 网络问题。逐项检查即可。

npm 安装 Claude Code 非常慢（超过 5 分钟）？

检查是否配置了国内镜像：npm config get registry（应显示 registry.npmmirror.com）。如果没有，执行 npm config set registry https://registry.npmmirror.com/ 后重试。

CC-Switch 打不开或闪退？

到 GitHub Release 重新下载最新版本，或尝试右键以管理员身份运行。

GitHub 页面打不开？

原因：GitHub 在国内可能访问不稳定。 解决：

  • 多刷新几次页面
  • 尝试使用不同的浏览器
  • 如果实在打不开，可以在网上搜索"CC-Switch 下载"，有些国内镜像站也提供了下载
